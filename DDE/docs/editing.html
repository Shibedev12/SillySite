Editing Music

So you're the kind who wants to make music, eh? thats good!

What you may see in the music section may confuse you, or it may not. who knows?

You may see this and be confused. Worry not, as it is not too hard to pull off.

Here's how it works.

  • What note do you want to use? lets say you want to use the up note.
  • You would type "u" into the music constant.
  • const music = ["u",]; if you have notes coming after that note, place a comma after the closing quote.
  • so an example would be this: const music = ["u", "d"]
  • A full list of the note types are as follows:
  • "u" up. the up note.
  • "d" down. the down note.
  • "l" left. the left note.
  • "r" right. the right note.

Makes sense? good! Thats pretty much it here.

Onto -

Delays

Very few of these, but they are needed for a functional setup, so follow along. I'll list the 2 main delays for you to edit.

The Start Delay

Description: if you need a pause before starting the notes in your music, you can!

How it works: The delay is in ms. so 1000 ms is 1 full second. 2000 is 2 seconds, and so on and so forth. make sense? Good. onto the next delay.

The Note Delay

Description: This is the delay between notes. you want it just right, or it will cause issues like note cutting. This delay is generally set at 900ms, or .90th of a second.

How it works: the delay is also in ms. 1000 ms is 1 second, and so on and so forth.

Now that you understand the delays, it is time for the extras. Quirks if you will.

Quirks

Not as few as the delays but there are enough to talk about them.

moveNoteToBottom

if you want to enable "freestyling", uncomment them. this will make it so that when the notes have reached the top, and are not moving, when the button is pressed, they return to the bottom.

in order to enable them, simply uncomment them, and vice versa.

Note Speed

controls the speed of the notes. by default it is set at 80ms. speeding it up is making it smaller, and increasing is making it bigger.
